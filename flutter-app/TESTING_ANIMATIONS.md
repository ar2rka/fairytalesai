# Тестирование анимаций онбординга

## Текущий вариант анимаций: Scale + Bounce (Вариант 3)

Реализован вариант **Scale + Bounce (игривый)**:
- Элементы увеличиваются с пружинящим эффектом (Curves.elasticOut)
- Иконки подпрыгивают при появлении
- Карточки появляются с легким отскоком
- Подходит для детского приложения - добавляет игривость и энергию
- Используются кривые Curves.elasticOut для пружинящего эффекта

## Способ 1: Визуальное тестирование (рекомендуется)

### Вариант A: Через обычный флоу приложения

1. **Запустите приложение:**
   ```bash
   flutter run
   ```

2. **Чтобы увидеть онбординг:**
   - Если вы уже авторизованы, нужно удалить профиль или детей из базы данных
   - Или используйте новый аккаунт
   - Или временно измените логику в `main.dart` (см. Вариант B)

3. **Проверьте анимации:**
   - Страница приветствия: иконка должна появиться с bounce эффектом
   - Страница функций: карточки должны появляться последовательно
   - Страница добавления ребенка: форма должна появиться с fade и slide
   - Страница интересов: чекбоксы должны появляться последовательно
   - Индикаторы прогресса: точки должны плавно изменять размер и цвет

### Вариант B: Прямой доступ к онбордингу (для тестирования)

Временно измените `main.dart` для прямого доступа к онбордингу:

```dart
// В файле lib/main.dart, замените строку 58:
home: const AuthWrapper(),

// На:
home: const OnboardingScreen(),
```

Затем запустите:
```bash
flutter run
```

**Не забудьте вернуть изменения обратно после тестирования!**

## Способ 2: Создание тестового экрана

Создайте отдельный тестовый экран для быстрого доступа:

```dart
// lib/presentation/screens/test_onboarding_screen.dart
import 'package:flutter/material.dart';
import 'onboarding_screen.dart';

class TestOnboardingScreen extends StatelessWidget {
  const TestOnboardingScreen({super.key});

  @override
  Widget build(BuildContext context) {
    return const OnboardingScreen();
  }
}
```

И временно используйте его в `main.dart`.

## Способ 3: Использование Flutter DevTools

1. **Запустите приложение с профилированием:**
   ```bash
   flutter run --profile
   ```

2. **Откройте Flutter DevTools:**
   - В консоли появится ссылка типа: `http://127.0.0.1:9100?uri=...`
   - Или откройте DevTools через VS Code/Android Studio

3. **Проверьте производительность анимаций:**
   - Performance tab → Record
   - Выполните онбординг
   - Остановите запись
   - Проверьте FPS (должен быть ~60 FPS)
   - Проверьте, нет ли jank (задержек)

---

## Способ 4: Замедление анимаций для детального просмотра

Для детального просмотра анимаций можно временно увеличить длительность:

В файлах онбординга найдите `AnimationController` и увеличьте `duration`:

```dart
// Например, в welcome_onboarding_page.dart:
_controller = AnimationController(
  duration: const Duration(milliseconds: 700), // Увеличьте до 2000-3000
  vsync: this,
);
```

## Советы по тестированию:

1. **Тестируйте на реальном устройстве** - анимации могут отличаться на эмуляторе
2. **Проверьте на разных размерах экрана** - iPhone SE, iPhone 14 Pro Max, iPad
3. **Проверьте производительность** - анимации должны быть плавными (60 FPS)
4. **Проверьте на медленном устройстве** - если есть доступ к старым устройствам
5. **Тестируйте в release режиме** - `flutter run --release` для более точной производительности
